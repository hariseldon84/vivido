**Patent #1 — Per-Creator Style Model (Compounding Intelligence Engine)**
ML system building a persistent, project-spanning style model per creator — hook structures, B-roll patterns, pacing, audio treatment, cut rhythms — across all projects.
- Novel: No NLE accumulates per-creator style data across sessions. All systems are stateless per project.

**Patent #2 — YouTube-Native Timeline Primitives**
Timeline architecture where YouTube metadata (Chapters, Cards, End Screens, Shorts cut-points) are first-class timeline objects — rendered and edited in the timeline, not added in YouTube Studio post-export.
- Novel: Every existing NLE treats YouTube metadata as an export concern, not a creative object.
- Claim: NLE timeline system comprising platform-specific interactive metadata objects that render in the timeline preview and generate platform-native export packages.

**Patent #3 — Cross-Project Hook Intelligence**
System that maps viewer retention data (YouTube Analytics API) back to specific timeline moments in the source project — closing the loop between post-publish analytics and pre-publish editing decisions.
- Novel: No tool links editing timeline data (pre-publish) with platform analytics (post-publish).
- Claim: Method for mapping viewer engagement metrics to temporal positions in source editing projects and generating editing recommendations for subsequent projects.
- File with Patent #1 (same underlying system).

**Patent #4 — Adaptive Multi-Platform Audio Normalization (Live Preview)**
Audio pipeline that applies platform-specific loudness normalization (YouTube/Shorts/Podcast/Reels) in real-time during playback — not only at export.
- Novel: All existing tools normalize at export. No NLE previews platform-normalized audio live.
- Claim: Real-time audio rendering pipeline in NLE that dynamically applies platform-specific loudness profiles during timeline playback without modifying source audio.

**Patent #5 — AI Cinematic Pre-Visualization from Raw Footage**
Generates a first-pass rough cut from raw footage using the creator's style model (Patent #1) — not generic excitement detection, but the creator's specific learned editorial grammar.
- Novel: Auto-cut tools use generic energy detection. No system uses a creator-specific style model as the editorial framework for rough-cut generation.
- Claim: Method for generating personalized rough-cut sequences from unedited footage by applying a creator-specific style model to evaluate and sequence candidate clips consistent with historical editorial decisions.

## Method 1: Jobs-to-be-Done Reframe

### The Four Hire Moments

**Hire Moment #1 — The Filmmaker-Creator**
- Functional job: Get from raw footage to published without leaving one creative headspace
- Emotional job: *Feel like a filmmaker, not a logistics coordinator*
- Social job: "This looks like a production company made it" — when it was just them
- Fires: FCP + DaVinci + Audition + Canva + YouTube Studio + VidIQ + Notion + Frame.io
- Hire trigger: "The fourth time I lost my creative thread switching apps, I knew I needed one thing that does all of this."

**Hire Moment #2 — The CapCut Refugee / Windows Creator**
- Functional job: One serious tool on Windows, looks pro, no Creative Cloud tax
- Emotional job: *Feel like a real creator, not a beginner using a free app*
- Social job: Stop being embarrassed to say what software they use
- Fires: CapCut + patchwork replacements
- Hire trigger: "CapCut was banned. Premiere felt like a spaceship. DaVinci was color science I didn't understand. I just want something powerful that's still mine."

**Hire Moment #3 — The Agency/Studio Lead**
- Functional job: One platform where the whole team edits, reviews, delivers
- Emotional job: *Feel like they're running a studio, not herding cats*
- Social job: Show clients a professional review flow, not "here's a Dropbox link"
- Fires: FCP/Premiere (per-seat chaos) + Frame.io + Dropbox + Notion + Slack
- Hire trigger: "My editor sent the wrong version to the client for the third time. I need one system of record."

**Hire Moment #4 — The First-Time Filmmaker (UNTAPPED)**
- Functional job: Make something cinematic without 6 months learning software
- Emotional job: *Feel like the idea in their head can actually become the thing on screen*
- Social job: Submit to a festival. Not be embarrassed.
- Fires: Nothing — this is a net-new user no existing tool has captured
- Hire trigger: "I opened three professional editors and they all felt designed for people who already knew how to use them."

### The Universal Hire Statement
> "I'm hiring Vivido to collapse the distance between the idea in my head and the thing the world sees — without losing my creative identity in the process."
